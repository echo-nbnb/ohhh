## 🐍 第二步：配置 Python 环境

本项目使用 **Anaconda** 管理 Python 环境，环境名为 `ohhh`，Python 版本为 **3.12**。

### 安装 Anaconda

前往 [https://www.anaconda.com/](https://www.anaconda.com/) 下载并安装 Anaconda。

> 安装时勾选 **"Add Anaconda to PATH"**（或安装完后手动配置环境变量）

### 创建并激活 conda 环境

打开 **Anaconda Prompt**（在开始菜单搜索），执行以下命令：

```bash
# 创建名为 ohhh 的虚拟环境，Python 版本 3.12
conda create -n ohhh python=3.12

# 激活环境
conda activate ohhh
```

激活成功后，命令行前缀会变为 `(ohhh)`，说明已进入正确环境。

> 📦 **关于依赖库**：项目所需的第三方库后续会补充到 `requirements.txt` 中，届时执行 `pip install -r requirements.txt` 即可一键安装。

## ⚠️ 重要警告：提交前必读

**请务必谨慎操作！直接 push 会覆盖所有人的本地代码！**

- `git push` 会**直接覆盖**远程仓库的代码
- 其他成员的本地代码**不会**自动更新
- 如果本地代码与远程冲突，可能导致部分修改丢失

**但是！代码可以恢复：**
- Git 有完整的版本历史，任何版本都可以恢复
- 如果误操作，可以回退到之前的提交：`git reset --hard <commit-id>`
- 提交记录不会消失，只是覆盖了当前状态

**建议流程：**
1. 每天开始工作前先 `git pull` 拉取最新代码
2. 修改完成后 `git add .` → `git commit -m "说明"`
3. 最后 `git push` 推送到 GitHub

### 项目概述

**寻麓千年色** 是一个以湖湘文化为底蕴的 **智能交互装置**。

观众通过在装置台面上**摆放实体模块**，摄像头实时捕捉模块的位置与种类信息，经由 AI 视觉识别与语言模型处理，最终在投影仪上呈现出**个性化的多模态故事内容**（音效、视频、文字、数字纪念品）。

整个体验链路兼顾**实时性**与**高质量内容生成**，形成沉浸式的文化叙事体验。

### 模块一：视觉定位 + ESP32识别（本地运行）

> **混合方案：摄像头定位模块位置，ESP32识别模块ID和触摸状态**

#### 1.1 视觉定位（摄像头）

```
实时视频帧流  →  YOLO 模型  →  模块坐标 / 数量
```

| 项目 | 内容 |
|------|------|
| **架构** | YOLO v8n / YOLO v5s |
| **参数量** | 1.5M ～ 3.5M |
| **运行位置** | 本地（电脑端） |
| **核心框架** | PyTorch + Ultralytics |
| **输出结果** | 模块坐标、数量（用于空间定位） |

**实现步骤：**

1. 采集并标注各类模块图片（简化版：只需区分"有模块/无模块"）
2. 基于 Ultralytics 框架训练 YOLO 模型
3. 接入摄像头实时视频流，逐帧推理
4. 输出模块在识别区内的坐标

#### 1.2 ESP32 模块识别

```
模块内置ESP32 + 触摸传感器  →  WiFi/蓝牙  →  ESP32 Hub  →  唯一ID + 触摸状态
```

| 项目 | 内容 |
|------|------|
| **通信方式** | WiFi UDP / 蓝牙 BLE |
| **每个模块** | ESP32 + 触摸传感器 + 唯一ID |
| **Hub功能** | 汇总所有模块信息，按需查询 |
| **输出结果** | 模块唯一ID、触摸状态、在线状态 |

**ESP32通信协议**（待定）：
- 模块上电后自动连接Hub
- 触摸事件实时上报
- 支持轮询查询当前在线模块

#### 1.3 数据融合

```
视觉定位（坐标）  +  ESP32（ID+触摸）  →  融合数据
                                           │
                    ┌──────────────────────┘
                    ↓
              每个模块的：位置 + ID + 触摸状态
```

### 模块二：AI 个性化叙事生成（RAG）

> **根据模块组合生成专属的湖大千年色叙事体验**

#### 2.1 整体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                         RAG 模块                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────┐     ┌─────────────┐     ┌─────────────────┐   │
│  │  知识库     │────▶│  检索引擎   │────▶│  本地小模型     │   │
│  │  (文化数据) │     │  (BM25)     │     │  (Ollama/待定)  │   │
│  └─────────────┘     └─────────────┘     └────────┬────────┘   │
│                                                   │             │
│  ┌───────────────────────────────────────────────┴─────────┐   │
│  │                      输出分层                          │   │
│  ├──────────────────────┬───────────────────────────────┤   │
│  │     实时侧           │        高质量侧               │   │
│  │  (每放置一个模块)    │       (开始生成时)            │   │
│  ├──────────────────────┼───────────────────────────────┤   │
│  │  • 元素一句话描述     │  • 完整文化上下文              │   │
│  │  • 连接关系类型      │  • 多元素组合解读              │   │
│  │  • 节点颜色/样式    │  • 打包发送给云端API           │   │
│  │  → Unity即时显示     │  → 云端生成图文叙事            │   │
│  └──────────────────────┴───────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

#### 2.2 输入输出定义

**输入（来自Unity或ESP32 Hub）：**

```python
# 实时侧输入（每放置/连接一个模块）
{
    "event": "module_placed",      # module_placed / module_connected / generation_start
    "module_id": "esp32_001",
    "module_type": "color",        # color / object / character
    "position": (320, 240),
    "connections": [               # 当前模块的所有连接
        {"to": "esp32_003", "type": "color_grant"},
    ],
    "all_modules": [...]           # 当前所有已放置模块
}

# 高质量侧输入（开始生成时）
{
    "event": "generation_start",
    "all_modules": [...],
    "all_connections": [...]
}
```

**实时侧输出（Python → Unity）：**

```python
{
    "module_id": "esp32_001",
    "entity": "岳麓绿",
    "description": "这是一种生命的颜色，代表着千年不息的传承。",  # 小模型润色
    "connections": [
        {"to": "esp32_003", "type": "color_grant", "style": "绿色光线"}
    ],
    "unity_data": {
        "node_color": "#2E7D32",
        "line_color": "#FFD700",     # 金色虚线=精神共鸣
        "line_style": "dashed"
    }
}
```

**高质量侧输出（Python → 云端API）：**

```python
{
    "api_payload": {
        "modules": [
            {"entity": "岳麓绿", "description": "...", "context": "..."},
            {"entity": "书院", "description": "...", "context": "..."},
            {"entity": "张栻", "description": "...", "context": "..."},
        ],
        "combinations": [...],
        "character_spirit": "传道济民",
        "prompt_template": "根据以下元素生成个性化叙事..."
    }
}
```

### 模块四：Unity 实时渲染与交互呈现

> **驱动投影仪，呈现沉浸式湖大叙事体验**

| 输入数据 | 输出效果 |
|---------|---------|
| 模块位置+ID（实时） | 场景中模块的虚拟对应物 |

#### 四幕交互流程

| **第一幕：择色** | 放置颜色桶 | 光圈绽放，空间染上颜色基调，音效响起 |
| **第二幕：筑景** | 放置物象块 | 元素生长，与环境融合，呈现独特视觉风格 |
| **第三幕：唤灵** | 放置人物块 | 光柱升起，人物精神与环境共鸣活化 |
| **第四幕：成色** | 完成组合 | 个性化叙事呈现，数字/实体纪念品生成 |

- 接收 Python 端通过 Socket 传递的结构化数据
- 实时更新场景中颜色、物象、人物的状态
- 投影呈现四幕叙事，体验结束后输出纪念品

## 🤝 协作规范

### Git 命令大全（直接复制使用）

#### 首次克隆仓库（仅执行一次）

```bash
git clone https://github.com/echo-nbnb/ohhh.git
cd ohhh
```

#### 每天开始工作前（拉取最新代码）

```bash
git checkout
git pull
```

#### 修改文件后，提交并推送到 GitHub

```bash
# 1. 查看当前修改状态
git status

# 2. 添加要提交的文件（. 表示全部，也可指定单个文件）
git add .

# 3. 提交并写明修改内容，格式：[模块名] 具体做了什么
git commit -m "[vision] 添加 YOLO 识别模块"

# 4. 推送到 GitHub（首次推送会要求输入用户名和 Token）
git push
```

> **Token 获取方式**：GitHub → 右上角头像 → Settings → Developer settings → Personal access tokens → Generate new token（勾选 repo 项）

#### 查看提交历史

```bash
git log --oneline
```

#### 创建新功能分支

```bash
git checkout -b feature/你的名字
```

#### 合并他人更新到当前分支

```bash
git pull
```

### 项目概述

**寻麓千年色** 是一个以湖湘文化为底蕴的 **智能交互装置**。

用户通过**手部动作**全程与投影画面交互——摄像头实时捕捉手部 21 个关键点，AI 识别手势意图（绘画、选择、浏览、确认），驱动四幕叙事体验。从选择颜色、手势绘画物象，到 AI 推荐历史人物、生成个性化精神画像，形成完整的沉浸式文化叙事体验。

整个体验链路兼顾**实时性**（逐帧手部追踪 <50ms 延迟）与**高质量内容生成**（RAG 检索 + LLM 归因 + 云端生图）。

### 核心概念

**"寻麓千年色"** —— 不是在找颜色，而是在寻找湖大千年精神中属于"我"的那一抹色彩。

- **寻**：探索、追问、对话
- **麓**：岳麓，根脉所在
- **千年**：从张栻到今天的精神传承
- **色**：不只是颜色，更是精神底色

**最终输出：一段专属于玩家的个性化叙事** —— AI 根据玩家的选择，为其编织的"精神自画像"。

## 🎭 交互设计

### 交互方式：纯手势驱动

用户全程通过**手部动作**与投影画面交互，无需物理模块。摄像头捕获手部 21 个关键点（MediaPipe），实时识别手势并驱动四幕叙事。

### 手势定义

#### 全局手势（任意阶段可用）

| 手势 | 识别方式 | 效果 |
|------|---------|------|
| 握拳挥动 | 五指握拳 + 手部移动 | 颜色在画作中扩散/晕染 |
| 五指张开 | 五指伸展 | 停止晕染 / 取消当前操作 |

#### 绘画手势（第二幕）

| 手势 | 识别方式 | 效果 |
|------|---------|------|
| 食指伸出 | 仅食指伸展，其余四指握起 | 进入绘画模式，食指尖为笔尖，轨迹实时渲染 |
| 握拳 | 五指全部握起 | 结束绘画 / 确认选中候选物象 |
| 五指张开 | 五指全部伸展 | 取消绘画，清空轨迹 |

#### 选择手势（第二幕/第三幕）

| 手势 | 识别方式 | 效果 |
|------|---------|------|
| 手悬停于候选区上方 | 手掌中心在目标区域内 | 目标高亮、放大、金色边框 |
| 握拳（悬停状态下） | 悬停 + 握拳 | 确认选中当前目标 |
| 五指张开（选中状态下） | 五指伸展 | 取消选中，返回候选列表 |

#### 轮盘手势（第三幕轮盘模式）

| 手势 | 识别方式 | 效果 |
|------|---------|------|
| 手左右滑动 | 手掌水平位移速度 > 阈值 | 轮盘卡片横向滚动，速度与手速正相关 |
| 手悬停 > 1秒 | 手部静止检测 + 计时 | 卡片放大，显示人物名 + 朝代 + 简介 |
| 握拳 | 悬停 + 握拳 | 确认选中人物 |
| 五指张开 | 五指伸展 | 退出轮盘，回到推荐阶段 |

### 四幕叙事流程

```
开场引入 → 第一幕:择色 → 第二幕:筑景 → 第三幕:唤灵 → 第四幕:成色 → 尾声
```

#### 开场引入
- 全黑缓缓亮起，湖大校门剪影水墨晕染浮现
- 泛黄邀请函飘落："岳麓千年，色隐其中。后来者，汝心何色？"
- AI 旁白欢迎玩家 → 摄像头检测到玩家 → 六个颜色桶虚影悬浮

#### 第一幕：择色——"我选择的底色"
- 手势悬停选择颜色（岳麓绿/书院红/西迁黄/湘江蓝/校徽金/墨色）
- 光圈绽放，颜色基调充满投影面
- 每种颜色有独立的 AI 旁白、画面意象和音效

#### 第二幕：筑景——"我看见的世界"
- 食指伸出进入绘画模式，指尖光点轨迹实时渲染
- 握拳结束绘画 → 轨迹栅格化 28x28 → CNN(QuickDraw MobileNet) 分类
- 345 类 QuickDraw 映射 88 物象 → 颜色上下文加权 → Top-3 候选浮现
- 悬停 + 握拳确认选中物象，五指张开放弃重画
- 可放置多个物象（建议 1-3 个），物象间可建立连接线

#### 第三幕：唤灵——"我听见的声音"
- **阶段一（推荐）**：RAG 检索颜色+物象 → LLM 排序 Top-3 人物卡片浮现
- 悬停预览人物简介 → 握拳确认选中 → 光柱冲天而起
- **阶段二（轮盘）**：五指张开拒绝推荐 → 进入轮盘模式
- 5 组 160 人物横向滚动浏览（古代先贤/近代湖湘/现代学人/校园角色/抽象意象）
- 选中人物后，AI 以人物第一人称独白，多人物可触发对话

#### 第四幕：成色——"这是属于我的千年色"
- 自由调整阶段：移动物象、建立连接、手势晕染
- 放置"确认章"触发最终生成
- RAG 检索 → 个性化叙事文本 → LLM 归因 → 云端 API 生图
- 最终画作呈现 + 精神画像叙事 + 二维码带走数字资产

> 完整情境细节（AI 旁白文本、色彩意象、人物独白等）见 [`proposal/interaction.md`](proposal/interaction.md)

### 物象识别管线

```
MediaPipe 食指指尖轨迹（点序列）
        ↓
轨迹栅格化：28x28 灰度图，笔画宽度 2px
        ↓
CNN 推理（QuickDraw 预训练 MobileNet，345 类）
        ↓
QuickDraw 类别 → 88 物象知识库映射
        ↓
颜色上下文加权（第一幕颜色对候选物象加权/降权）
        ↓
返回 Top-3 候选物象（Top-1 居中高亮，金色边框）
```

### 人物推荐管线

```
第一幕颜色 + 第二幕物象
        ↓
RAG 检索人物知识库（160 条目，BM25 + 语义匹配）
        ↓
LLM 排序 Top-3 推荐人物
        ↓
用户确认 / 拒绝后进入轮盘自选（5 组 160 人物）
```

### 连接类型矩阵

| 连接类型 | 组合 | 视觉样式 | 叙事逻辑 |
|---------|------|---------|---------|
| 精神共鸣 | 颜色 to 颜色 | 金色虚线 | 情感混合，产生新内涵 |
| 精神赋予 | 颜色 to 物象 | 物象对应色光线 | 颜色赋予物象情感解读 |
| 人格染色 | 人物 to 颜色 | 人物对应色光线 | 人物特质影响颜色解读 |
| 场景进入 | 物象 to 人物 | 白色实线 | 激活历史典故 |
| 意境叠加 | 物象 to 物象 | 白色实线 | 意象联想，产生新场景 |
| 精神对话 | 人物 to 人物 | 青色实线 | 对话、师承、对比关系 |

### 画作风格决策

| 人物年代 | 推荐风格 | 说明 |
|---------|---------|------|
| 古代（朱熹等） | 水墨写意 | 符合历史质感 |
| 近现代（李达等） | 老照片实景 | 穿越感 |
| 混合年代 | 穿越风格 | 古今同框 |

### 叙事生成逻辑

```
用户选择（颜色+物象+人物+连接）
        ↓
构建连接关系图 → 提取核心元素（连接数最多）
        ↓ （无法提取时）
按连接类型优先级保底：
  spirit_dialogue > personality_dye > scene_entry
  > color_grant > scene_combination > spiritual_resonance
        ↓ （复杂场景降级）
LLM 自由发挥
        ↓
生成个性化叙事 + 生图提示词 → 云端 API 生成最终画作
```

### 模块一：手势识别与追踪（本地运行）

> **MediaPipe Hand Landmarker：21 个手部关键点，逐帧追踪**

| 项目 | 内容 |
|------|------|
| **模型** | MediaPipe Hand Landmarker (Google) |
| **输出** | 21 个归一化关键点坐标 (x, y, z) |
| **运行位置** | 本地（Python） |
| **帧率** | 30fps（摄像头） / 无上限（no-display 模式） |

**手势识别逻辑**（`vision/gesture_state_machine.py`）：

| 状态模式 | 说明 |
|---------|------|
| GLOBAL | 全局空闲，检测握拳晕染 / 五指张开 |
| DRAWING | 食指绘画模式，追踪食指尖轨迹 |
| CANDIDATE | Top-3 候选物象选择，检测悬停 + 握拳确认 |
| CHAR_RECOMMEND | 人物推荐卡片，检测悬停 + 握拳确认 / 张开拒绝 |
| CHAR_WHEEL | 人物轮盘，检测水平滑动 + 悬停 + 握拳选择 |

#### 4.3 延迟优化

| 措施 | 效果 |
|------|------|
| TCP_NODELAY（禁用 Nagle 算法） | 减少 ~200ms 延迟 |
| 摄像头帧缓冲排空（每帧 4 次 read） | 消除帧堆积 |
| 预创建对象池（无 GC 分配） | 消除逐帧 GC 卡顿 |
| no-display 模式（test_integrated_fast.py） | 去掉 OpenCV 显示开销 |

### Git 协作规范

#### 首次克隆仓库（仅执行一次）

```bash
git clone https://github.com/echo-nbnb/ohhh.git
cd ohhh
```

#### 每天开始工作前（拉取最新代码）

```bash
git checkout
git pull
```

#### 修改文件后，提交并推送到 GitHub

```bash
# 1. 查看当前修改状态
git status

# 2. 添加要提交的文件（. 表示全部，也可指定单个文件）
git add .

# 3. 提交并写明修改内容，格式：[模块名] 具体做了什么
git commit -m "[vision] 添加 YOLO 识别模块"

# 4. 推送到 GitHub（首次推送会要求输入用户名和 Token）
git push
```

> **Token 获取方式**：GitHub → 右上角头像 → Settings → Developer settings → Personal access tokens → Generate new token（勾选 repo 项）

#### 查看提交历史

```bash
git log --oneline
```

#### 创建新功能分支

```bash
git checkout -b feature/你的名字
```

#### 合并他人更新到当前分支

```bash
git pull
```

### 快速开始

#### 第一步：从 GitHub 下载项目

**方式一：直接下载压缩包**
1. 打开本仓库页面
2. 点击 **`Code`** → **`Download ZIP`**
3. 解压到你想放的文件夹

**方式二：使用 Git 克隆（推荐）**

前往 [https://git-scm.com/](https://git-scm.com/) 下载并安装 Git

```bash
git clone https://github.com/echo-nbnb/ohhh.git
cd ohhh
```

#### 第二步：配置 Python 环境

本项目使用 **Anaconda** 管理 Python 环境，环境名为 `ohhh`，Python 版本为 **3.12**。

```bash
conda create -n ohhh python=3.12
conda activate ohhh
pip install -r requirements.txt
```

#### 第三步：在 PyCharm 中打开项目

1. 启动 PyCharm，点击 **Open**，选择项目文件夹
2. 配置 Python 解释器：`Settings` → `Project` → `Python Interpreter` → `Add Interpreter` → `Conda Environment` → `Use existing environment` → 选择 `ohhh`

## ⚠️ 重要警告

**请务必谨慎操作！直接 push 会覆盖所有人的本地代码！**

- `git push` 会**直接覆盖**远程仓库的代码
- 其他成员的本地代码**不会**自动更新

**建议流程：**
1. 每天开始工作前先 `git pull` 拉取最新代码
2. 修改完成后 `git add .` → `git commit -m "说明"`
3. 最后 `git push` 推送到 GitHub

## 🚀 第一步：从 GitHub 下载项目

### 方式一：直接下载压缩包（最简单，不推荐长期使用）

1. 打开本仓库页面
2. 点击右上角绿色按钮 **`Code`**

### 方式二：使用 Git 克隆（推荐）

#### 第一步：安装 Git

前往 [https://git-scm.com/](https://git-scm.com/) 下载并安装 Git，一路点 Next 即可。

安装完成后，在任意位置右键，如果看到 **Git Bash Here** 说明安装成功。

#### 第二步：克隆仓库

1. 打开 **Git Bash** 或 Windows 的 **命令提示符（cmd）**
2. 进入你想存放项目的文件夹，例如：

```bash
cd D:/MyProjects
```

3. 执行以下命令克隆仓库：

```bash
git clone https://github.com/你的用户名/仓库名.git
```

> 把上面的地址替换成本仓库实际的 GitHub 地址（在仓库页面点 Code 按钮复制）

4. 克隆完成后进入项目文件夹：

```bash
cd 仓库名
```

#### 第三步：后续更新代码

当其他成员更新了代码，你可以在项目文件夹内执行以下命令拉取最新内容：

```bash
git pull
```

## 🐍 第二步：配置 Python 环境

本项目使用 **Anaconda** 管理 Python 环境，环境名为 `ohhh`，Python 版本为 **3.12**。

### 安装 Anaconda

前往 [https://www.anaconda.com/](https://www.anaconda.com/) 下载并安装 Anaconda。

> 安装时勾选 **"Add Anaconda to PATH"**（或安装完后手动配置环境变量）

### 创建并激活 conda 环境

打开 **Anaconda Prompt**（在开始菜单搜索），执行以下命令：

```bash
# 创建名为 ohhh 的虚拟环境，Python 版本 3.12
conda create -n ohhh python=3.12

# 激活环境
conda activate ohhh
```

激活成功后，命令行前缀会变为 `(ohhh)`，说明已进入正确环境。

> 📦 **关于依赖库**：项目所需的第三方库后续会补充到 `requirements.txt` 中，届时执行 `pip install -r requirements.txt` 即可一键安装。

## 💻 第三步：在 PyCharm 中打开项目

### 安装 PyCharm

前往 [https://www.jetbrains.com/pycharm/](https://www.jetbrains.com/pycharm/) 下载 **Community（社区免费版）** 安装即可。

### 打开项目

1. 启动 PyCharm
2. 点击 **Open**
3. 选择你克隆/解压的项目文件夹，点击 **OK**

### 配置 Python 解释器（关键步骤！）

> 必须将解释器设置为 conda 的 `ohhh` 环境，否则代码无法运行。

1. 打开 PyCharm，点击右下角的解释器版本显示（或进入菜单 `File` → `Settings`）
2. 依次点击：`Settings` → `Project: 项目名` → `Python Interpreter`
3. 点击右侧的 **齿轮图标** → **Add Interpreter** → **Add Local Interpreter**
4. 左侧选择 **Conda Environment**
5. 选择 **Use existing environment**
6. 在下拉菜单中选择 **ohhh**
7. 点击 **OK** 确认

配置完成后，右下角应显示 `Python 3.12 (ohhh)`。

## ⚠️ 重要警告：提交前必读

**请务必谨慎操作！直接 push 会覆盖所有人的本地代码！**

- `git push` 会**直接覆盖**远程仓库的代码
- 其他成员的本地代码**不会**自动更新
- 如果本地代码与远程冲突，可能导致部分修改丢失

**但是！代码可以恢复：**
- Git 有完整的版本历史，任何版本都可以恢复
- 如果误操作，可以回退到之前的提交：`git reset --hard <commit-id>`
- 提交记录不会消失，只是覆盖了当前状态

**建议流程：**
1. 每天开始工作前先 `git pull` 拉取最新代码
2. 修改完成后 `git add .` → `git commit -m "说明"`
3. 最后 `git push` 推送到 GitHub

## 📤 第四步：如何修改并上传文件

### 在 PyCharm 中修改代码

直接在左侧文件树中找到你要编辑的文件，双击打开，修改保存（`Ctrl+S`）即可。

### 提交并推送到 GitHub（直接推送到 main 分支）

PyCharm 内置了 Git 工具，无需手动敲命令。

#### 方式一：使用 PyCharm 图形界面（推荐新手）

1. 修改文件后，点击顶部菜单 `Git` → `Commit`（或按 `Ctrl+K`）
2. 在左侧勾选你修改的文件
3. 在下方输入**提交说明**（必填！请写清楚你改了什么，例如：`添加YOLO识别模块基础代码`）
4. 点击 **Commit and Push**
5. 弹窗中点击 **Push** 确认推送

> ⚠️ **警告**：推送前请确认你修改的内容是正确的！推送到 main 后会直接影响其他所有人！

#### 方式二：使用命令行（进阶）

在 PyCharm 底部打开 **Terminal**，依次执行：

```bash
# 查看修改了哪些文件
git status

# 将修改的文件加入暂存区（. 表示全部文件）
git add .

# 提交并写说明
git commit -m "你的修改说明"

# 推送到 GitHub（直接推送到 main 分支）
git push
```

> ⚠️ **再次警告**：`git push` 会直接覆盖远程代码，请确认你的修改是正确的！

### ⚠️ 协作注意事项

- **每次开始工作前先 `git pull`**，确保拿到最新代码
- **提交说明要写清楚**，方便其他成员理解
- 如遇代码冲突（conflict）请及时告知负责人
- **不要创建新分支，直接推送到 main 即可**

### 项目概述

**寻麓千年色** 是一个以湖湘文化为底蕴的 **智能交互装置**。

观众通过在装置台面上**摆放实体模块**，摄像头实时捕捉模块的位置与种类信息，经由 AI 视觉识别与语言模型处理，最终在投影仪上呈现出**个性化的多模态故事内容**（音效、视频、文字、数字纪念品）。

整个体验链路兼顾**实时性**与**高质量内容生成**，形成沉浸式的文化叙事体验。

```

                    │                 │
                    │  ┌───────────┐  │         ┌──────────────┐
                    │  │ 模块位置   │──┼─────→   │  交互动效音效 │
                    │  │ 模块种类   │  │         │  个性化故事   │
                    │  │ 开始生成   │  │  (高质量)│  数字纪念品   │
                    │  └───────────┘  │         └──────────────┘
                    │       ↕         │
                    │    数据库        │
                    └────────┬────────┘
                             │
                          云  端
                    ┌────────┴────────┐
                    │   云端大模型     │→  多模态故事信息
                    └─────────────────┘
```

> **紫色路径** = 实时性优先（快速响应）
> **绿色路径** = 高质量优先（精细内容）

### 模块一：AI 视觉识别（本地运行）

> **负责将摄像头画面解析为结构化数据**

```
实时视频帧流  →  YOLO 模型  →  类别 / 坐标 / 置信度
```

| 项目 | 内容 |
|------|------|
| **架构** | YOLO v8n / YOLO v5s |
| **参数量** | 1.5M ～ 3.5M |
| **运行位置** | 本地（电脑端） |
| **核心框架** | PyTorch + Ultralytics |
| **训练数据** | 模块图片 + 人物图片 + 公开数据集 |
| **输出结果** | 模块类别、模块坐标、置信度 |

**实现步骤：**

1. 采集并标注各类模块图片，构建训练数据集
2. 基于 Ultralytics 框架训练 YOLO 模型
3. 接入摄像头实时视频流，逐帧推理
4. 将识别结果（类别、坐标、置信度）格式化输出，传递给下游模块

---

### 模块二：AI 检索生成（本地运行）

> **根据模块信息检索匹配文化内容，生成结构化指令**

```
模块信息  →  RAG（本地数据集 + 轻量LLM）  →  简要指令 / 完整准确信息
```

| 项目 | 内容 |
|------|------|
| **架构** | 检索算法（BM25 等）+ 轻量 LLM |
| **参数量** | 1B ～ 2B |
| **运行位置** | 本地（电脑端） |
| **核心技术** | RAG（检索增强生成） |
| **数据来源** | 本地文化知识数据库 |

**实现步骤：**

1. 构建湖湘文化相关本地知识数据库（文字、图片、音频索引）
2. 接收 YOLO 输出的模块信息
3. 使用 BM25 等检索算法从数据库中召回相关内容
4. 调用轻量 LLM（1-2B）对检索内容进行整合，生成：
   - 给 Unity 使用的**简要控制指令**
   - 给故事生成模块使用的**完整、准确的元素信息**

---

### 模块三：多模态故事生成（云端运行）

> **融合风格、人物、元素信息，生成完整的多模态故事内容**

```
风格信息 ┐
人物信息 ├→  规范要求  →  多 Agent 协作  →  多模态故事信息
元素信息 ┘
```

| 项目 | 内容 |
|------|------|
| **架构** | 顶层架构（入口、调度、融合）+ 基座模型 |
| **参数量** | 按需选择（云端调用） |
| **运行位置** | 云端 |
| **Agent 类型** | 调度Agent / 文本Agent / 视觉Agent / 音频Agent / 融合Agent |

**实现步骤：**

1. 接收本地 RAG 模块生成的完整元素信息
2. **调度 Agent** 将任务分发给各专项 Agent
   - **文本 Agent** → 生成故事文案
   - **视觉 Agent** → 生成/匹配图像视频内容
   - **音频 Agent** → 生成/匹配音效音乐
3. **融合 Agent** 将三路内容整合为统一的多模态故事包
4. 将内容推送回本地，由 Unity 负责最终呈现

---

### 模块四：Unity 实时渲染与交互呈现

> **统一调度所有数据，驱动投影仪输出**

| 输入数据 | 输出效果 |
|---------|---------|
| 模块位置数据（实时） | 交互动效音效 |
| 模块种类数据（实时） | 个性化故事内容 |
| 多模态故事信息（云端） | 数字纪念品生成 |

- 接收 Python 端通过 Socket / 本地通信传递的结构化数据
- 根据数据实时更新场景中模块的位置与状态
- 触发对应的动效、音效与投影内容
- 高质量内容（故事、纪念品）在云端数据就绪后异步加载呈现

## 🤝 协作规范

### Git 命令大全（直接复制使用）

#### 首次克隆仓库（仅执行一次）

```bash
git clone https://github.com/echo-nbnb/ohhh.git
cd ohhh
```

#### 每天开始工作前（拉取最新代码）

```bash
git checkout dev
git pull origin dev
```

#### 修改文件后，提交并推送到 GitHub

```bash
# 1. 查看当前修改状态
git status

# 2. 添加要提交的文件（. 表示全部，也可指定单个文件）
git add .

# 3. 提交并写明修改内容，格式：[模块名] 具体做了什么
git commit -m "[vision] 添加 YOLO 识别模块"

# 4. 推送到 GitHub（首次推送会要求输入用户名和 Token）
git push
```

> **Token 获取方式**：GitHub → 右上角头像 → Settings → Developer settings → Personal access tokens → Generate new token（勾选 repo 项）

#### 查看提交历史

```bash
git log --oneline
```

#### 创建新功能分支

```bash
git checkout -b feature/你的名字
```

#### 合并他人更新到当前分支

```bash
git pull origin dev
```

### Git 提交说明规范

请按照以下格式书写提交说明：

```
[模块名] 具体做了什么

例如：
[vision] 添加 YOLO v8n 模型训练脚本
[rag] 完成 BM25 检索基础实现
[story] 修复文本 Agent 输出格式问题
```

| 分支名 | 用途 |
|--------|------|
| `main` | 稳定版本，可运行的代码 |
| `dev` | 日常开发分支 |
| `feature/你的名字` | 个人开发新功能时使用 |

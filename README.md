### 项目概述

**寻麓千年色** 是一个以湖湘文化为底蕴的 **智能交互装置**。

观众通过在装置台面上**摆放实体模块**，摄像头实时捕捉模块的位置与种类信息，经由 AI 视觉识别与语言模型处理，最终在投影仪上呈现出**个性化的多模态故事内容**（音效、视频、文字、数字纪念品）。

整个体验链路兼顾**实时性**与**高质量内容生成**，形成沉浸式的文化叙事体验。

### 核心概念

**"寻麓千年色"** —— 不是在找颜色，而是在寻找湖大千年精神中属于"我"的那一抹色彩。

- **寻**：探索、追问、对话
- **麓**：岳麓，根脉所在
- **千年**：从张栻到今天的精神传承
- **色**：不只是颜色，更是精神底色

**最终输出：一段专属于玩家的个性化叙事** —— AI根据玩家的选择，为其编织的"精神自画像"。

## 🎭 交互设计

### 交互流程

```
1. 放置颜色块 → 奠定画作基调
2. 放置物象块 → 构建场景意境
3. 放置人物块 → 激活精神叙事
4. 任意阶段：手势挥动 → 晕染颜色
5. 任意阶段：触摸连接 → 建立模块间连线
```

### 手势识别

| 手势 | 效果 |
|------|------|
| 手握拳挥动 | 颜色在画作中扩散/晕染 |
| 手张开 | 颜色保持当前状态（停止晕染） |

### 模块放置顺序

用户跟随剧情流（颜色 → 物象 → 人物），但连线可在任意阶段进行。

---

### 节点连接类型矩阵

| 连接类型 | 组合 | 视觉样式 | 画作变化 | 叙事逻辑 |
|---------|------|---------|---------|---------|
| **精神共鸣** | 颜色 ↔ 颜色 | 金色虚线 | 两色交融、渐变 | 情感混合，产生新内涵 |
| **精神赋予** | 颜色 ↔ 物象 | 物象对应色光线 | 物象被染色 | 颜色赋予物象情感解读 |
| **人格染色** | 人物 ↔ 颜色 | 人物对应色光线 | 人物被染色 | 人物特质影响颜色解读 |
| **场景进入** | 物象 ↔ 人物 | 白色实线 | 人物光柱矗立于物象中 | 激活历史典故 |
| **意境叠加** | 物象 ↔ 物象 | 白色实线 | 两物并置、空间叠加 | 意象联想，产生新场景 |
| **精神对话** | 人物 ↔ 人物 | 青色实线 | 双人精神光柱 | 对话、师承、对比关系 |

### 同类连接详解

**颜色 ↔ 颜色（spiritual_resonance）**
- 🔴 红 + 🟢 绿 → 活力/生命力
- 🔴 红 + 🔵 蓝 → 神秘/深沉
- 🟢 绿 + 🔵 蓝 → 宁静/和谐

**物象 ↔ 物象（scene_combination）**
- 🏛️ 书院 + 🌊 湘江 → "书院的钟声回荡在湘江之上"
- 🏛️ 书院 + ⛰️ 岳麓山 → "人文与山水交融"
- 🌊 湘江 + ⛰️ 岳麓山 → "山水画卷"

**人物 ↔ 人物（spirit_dialogue）**
- 👤 张栻 + 👤 朱熹 → 理学对话
- 👤 曾国藩 + 👤 左宗棠 → 湘军将帅

### 个性化叙事生成逻辑（方案A：连接驱动叙事 + 核心元素提取）

> **第一版**，后续迭代

#### 核心思路

```
用户的所有选择（颜色+物象+人物+连接）
           ↓
    构建连接关系图
           ↓
    方案2：提取核心元素（连接数最多的模块）
           ↓
    无法提取 → 方案1：按连接类型优先级保底
           ↓
    方案D：LLM完全自由发挥（降级）
           ↓
    生成个性化叙事
```

#### 方案2：核心元素提取

核心元素 = 连接数最多的模块

```
# 示例：
# 连接：朱熹↔李达(1) + 朱熹↔湘江蓝(2) + 李达↔书卷(3) + 湘江蓝↔墨色(4)
# → 连接数：朱熹=2，李达=2，湘江蓝=2，墨色=1，书卷=1
# → 核心元素：湘江蓝（被连接最多）
# → 叙事主线：围绕"湘江蓝"展开，它是所有连接的枢纽
```

#### 方案1保底：连接类型优先级

```
priority_order = [
    'spirit_dialogue',      # 人物↔人物
    'personality_dye',      # 颜色↔人物
    'scene_entry',           # 物象↔人物
    'color_grant',          # 颜色↔物象
    'scene_combination',     # 物象↔物象
    'spiritual_resonance'    # 颜色↔颜色
]
```

#### 方案D降级

当连接数≥5且无法提取核心元素时，LLM完全自由发挥。

#### 流程图

```
用户选择 + 连接
       ↓
连接数统计
       ↓
    ┌──────┴──────┐
    ↓             ↓
有核心元素      无核心元素
(连接数相同)    ↓
    ↓         方案1保底：
方案2：       按连接类型优先级
围绕核心展开     ↓
    ↓         叙事生成
叙事生成        ↓
    ↓         ↓
    └──────┬──────┘
           ↓
    连接数≥5且复杂？
           ↓
      ┌────┴────┐
      ↓         ↓
     是         否
      ↓         ↓
   方案D降级   输出叙事
```

### 模块一：视觉定位 + ESP32识别（本地运行）

> **混合方案：摄像头定位模块位置，ESP32识别模块ID和触摸状态**

#### 1.1 视觉定位（摄像头）

```
实时视频帧流  →  YOLO 模型  →  模块坐标 / 数量
```

| 项目 | 内容 |
|------|------|
| **架构** | YOLO v8n / YOLO v5s |
| **参数量** | 1.5M ～ 3.5M |
| **运行位置** | 本地（电脑端） |
| **核心框架** | PyTorch + Ultralytics |
| **输出结果** | 模块坐标、数量（用于空间定位） |

#### 1.2 ESP32 模块识别

```
模块内置ESP32 + 触摸传感器  →  WiFi/蓝牙  →  ESP32 Hub  →  唯一ID + 触摸状态
```

| 项目 | 内容 |
|------|------|
| **通信方式** | WiFi UDP / 蓝牙 BLE |
| **每个模块** | ESP32 + 触摸传感器 + 唯一ID |
| **Hub功能** | 汇总所有模块信息，按需查询 |
| **输出结果** | 模块唯一ID、触摸状态、在线状态 |

#### 1.3 数据融合

```
视觉定位（坐标）  +  ESP32（ID+触摸）  →  融合数据
                                           │
                    ┌──────────────────────┘
                    ↓
              每个模块的：位置 + ID + 触摸状态
```

### 输入数据 → 生图提示词 转换逻辑（方案三：LLM归因）

> **第一版**，后续迭代

#### 核心逻辑

```
用户组合（颜色+物象+人物+连接）
       ↓
   LLM 阅读理解
       ↓
   生成完整生图提示词
       ↓
   发送到云端 API（阿里云百炼）
```

#### 流程详解

```
1. 输入示例：
   湘江蓝 + 墨色 + 书卷 + 星辰 + 朱熹 + 李达
   连接：朱熹 ↔ 李达（spirit_dialogue）

2. LLM 归因处理：
   将用户组合作为上下文，让 LLM 生成完整生图提示词

3. 输出：
   Stable Diffusion 英文提示词 → 阿里云百炼API → 返回画作
```

#### 实现代码

```python
def generate_prompt(user_input):
    """
    user_input = {
        "colors": ["湘江蓝", "墨色"],
        "objects": ["书卷", "星辰"],
        "characters": ["朱熹", "李达"],
        "connections": [("朱熹", "李达", "spirit_dialogue")]
    }
    """

    prompt = f"""根据以下湖大千年色组合，生成一幅 Stable Diffusion 画作描述：

颜色：{user_input['colors']}
物象：{user_input['objects']}
人物：{user_input['characters']}
连接：{user_input['connections']}

要求：
- 风格：穿越感，水墨画 + 老照片实景混合
- 人物：保留两个人影，远距离呈现
- 底图：颜色晕染成水墨背景
- 叙事元素融入画面

输出：一段英文提示词，200词以内，用于 text-to-image 生成"""

    response = ollama.chat(model="llama3", messages=[
        {"role": "user", "content": prompt}
    ])

    return response['message']['content']
```

#### 风格判断（融入 LLM 归因）

- 古代人物（朱熹等）→ 水墨写意风格
- 近现代人物（李达等）→ 老照片实景风格
- 混合年代 → 穿越风格，古今同框

#### 优点

- 语义最通顺，叙事性强
- 自动处理元素间关系
- 可解释性强

#### 缺点

- 需要调用 LLM，有延迟
- LLM 输出质量不稳定
- 两步都依赖云端，失败点增多

#### 失败处理

- LLM 输出不稳定 → 降级到模板填充（方案一）
- 云端 API 超时 → 返回"画作生成中，请稍候"

#### 待迭代项

- [ ] 优化 prompt 模板，提高 LLM 输出稳定性
- [ ] 增加风格判断逻辑的准确性
- [ ] 测试不同云端 API 的效果
